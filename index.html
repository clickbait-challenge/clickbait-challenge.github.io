Task Description

We invite you to participate in our challenge on the detection of clickbait posts in social media. Clickbait refers to social media posts designed to entice its readers into clicking an accompanying link at the expense of being informative and objective. More on clickbait.
The task of the challenge is to develop a classifier that rates how click baiting a social media post is. For each social media post to be judged, the content of the post itself as well as the main content of the linked target web page are provided as JSON-Object in our datasets.

Classifiers have to output a clickbait score in the range [0,1], where a value of 1.0 denotes that a post is heavily click baiting. 


{"id": "608999590243741697", "clickbaitScore": 1.0}

Performance is measured against a crowd-sourced test set. Every post in the training and test sets have been assessed on a 4-point scale [0, 0.3, 0.66, 1] by 10 annotators.

As primary evaluation metric Mean Squared Error (MSE) with respect to the mean judgments of the annotators is used. For informational purposes, we compute further evaluation measures such as Median Absolute Error (MedAE) or the Receiver Operating Characteristic (ROC) with respect to the mode of the judgments, as well as the runtime of the software.

How To Participate

  1. Register for the challenge to get a TIRA virtual machine.
  2. Develop and train a clickbait classifier on the training data.
  3. Deploy the trained classifier on the TIRA virtual machine assigned to you.
  1. Use tira.io to self-evaluate the deployed classifier on the test set.
  2. Write and submit a paper that describes how you approached the task.
  3. Pesent your approach at the workshop.

Related Work

  • Wikipedia. Clickbait --- Wikipedia, The Free Encyclopedia. 2017.

    Clickbait tweets typically aim to exploit the "curiosity gap", providing just enough information to make readers curious, but not enough to satisfy their curiosity without clicking through to the linked content.

  • Alex Peysakhovich and Kristin Hendrix. News Feed FYI: Further Reducing Clickbait in Feed. In Facebook newsroom, August, 2016.

    A tweet is Clickbait if (1) the tweet withholds information required to understand what the content of the article is; and if (2) the tweet exaggerates the article to create misleading expectations for the reader.

  • Ricardo Bilton. The many different ways publishers define 'clickbait'. In digiday UK, November, 2014.

    Clickbait is saying "this town" or "this state" or "this celebrity" instead of saying Los Angeles or Colorado or Justin Timberlake. It's over-promising and under-delivering. It's leaving out the one crucial piece of information the reader may want to know.

  • Martin Potthast, Sebastian Köpsel, Benno Stein, and Matthias Hagen. Clickbait Detection. In Advances in Information Retrieval (ECIR 16), March 2016. [bib]

    This paper presents the first machine learning approach to clickbait detection: the goal is to identify messages in a social stream that are designed to exploit cognitive biases to increase the likelihood of readers clicking an accompanying link.

Datasets

Over the course of the shared task, three datasets are going to be released. Each dataset consists of up to four files. Except the media archive, which is a zip file containing the images referenced in the posts, all files are valid JSON-Arrays. Each Object in the JSON-Array provides specific information about a specific post. The posts-file contains information about the post itself, the targets file contains information about the main content of the article advertised in the post. The truth file contains the crowd-sourced clickbait scores. Have a look at the dataset schema file that lists all the available fields for a post.

Software Submission

We use the Evaluation as a Service platform TIRA to evaluate the performance of your classifier. TIRA requires that your classifier can be executed with two arguments for input and output directories via a command line call. The E.g., the syntax could be:

 > myClassifier -i path/to/input/directory -o path/to/output/directory
example command line call for tira.io

At runtime, the input directory contains the dataset (i.e. posts and targets files plus media archive) your classifier has to process. The predictions of your classifier should be written into file called results.json into the given output directory. The results.json file should contain a valid JSON-Array, and each element of the array should be an Object holding the id and clickbait-score of a post (cf. the dataset schema file).

[
{"id": "608999590243741697", "clickbaitScore": 1.0},
{"id": "609408598704128000", "clickbaitScore": 0.25},
...
]
results.json

We will ask you to deploy your software onto a virtual machine that will be made accessible to you after registration. You can choose freely among the available programming languages and among the operating systems Microsoft Windows and Ubuntu. You will be able to reach the virtual machine via ssh and via remote desktop. More information about how to access the virtual machines can be found in the user guide below:

Virtual Machine User Guide »

Once deployed on your virtual machine, we ask you to access TIRA at www.tira.io, where you can self-evaluate your software on the test data.
